gated-info — Google Setup Guide

One service account — access to Drive, Sheets, Docs, BigQuery, Calendar.
Gmail needs OAuth2 (separate section below).

Steps

  1. Get a Google Cloud Project
  2. Create Service Account
  3. Enable APIs
  4. Grant Access
  5. Connect to gated-info

Step 0: Get a Google Cloud Project

Everything in Google Cloud lives inside a "project". You need one. It's free.

Already have a project? Skip to Step 1. Your project ID is in the URL when you open the Console:

console.cloud.google.com/home/dashboard?project=YOUR_PROJECT_ID

Don't have a project?

  1. Open Create a Project
    • First time? You'll see "Get started for free" — click it, agree to terms. No credit card needed.
  2. Project name: anything (e.g. my-tools, personal-mcp)
  3. Organization: leave "No organization" for personal Gmail. Workspace auto-selects — that's fine.
  4. Click Create. Wait 10 seconds.

Your Project ID looks like my-tools-438209 (name + random number). You'll paste it into commands below.

Where to find Project ID: Console top bar → click project dropdown → see "ID" column. Or in the URL after ?project=.
Free tier: Drive, Sheets, Docs APIs — free. BigQuery — 1 TB/month queries free. No billing needed for basic usage.

Step 1: Create Service Account

A service account is a "robot user" — it has an email, can access files and APIs. Never expires, no re-login.

Option A: Via Console (no tools needed)

  1. Open Service Accounts
  2. Make sure your project is selected in the top dropdown
  3. Click "+ Create Service Account"
    • Name: gated-info (or anything)
    • ID auto-fills (becomes the email)
    • Click Create and Continue
    • "Grant access to project" — skip for now, click Done
  4. You'll see the SA with email like: gated-info@your-project-id.iam.gserviceaccount.com
  5. Click on it → Keys tab → Add Key → Create new key → JSON → Create
  6. A .json file downloads — this is your key. Keep it safe.

Option B: Via gcloud CLI

gcloud iam service-accounts create gated-info --display-name="gated-info MCP"

Download the key (replace YOUR_PROJECT_ID):

gcloud iam service-accounts keys create ~/Downloads/gated-info-key.json --iam-account=gated-info@YOUR_PROJECT_ID.iam.gserviceaccount.com

Don't have gcloud? Option A does everything. If you want it: Install guide.

Step 2: Enable APIs

Enable only what you need. Each is a toggle.

Option A: Via CLI

Core (Drive / Sheets / Docs):

gcloud services enable drive.googleapis.com
gcloud services enable sheets.googleapis.com
gcloud services enable docs.googleapis.com

BigQuery:

gcloud services enable bigquery.googleapis.com

Gmail (requires OAuth — see Gmail section):

gcloud services enable gmail.googleapis.com

Calendar:

gcloud services enable calendar-json.googleapis.com

Option B: Via Console

  1. Open APIs & Services → Library
  2. Search for each API → click → Enable

Step 3: Grant Access

Google Drive / Sheets / Docs

Share folders or files with the SA email:

  1. In Google Drive, right-click folder → Share
  2. Paste: gated-info@YOUR_PROJECT.iam.gserviceaccount.com
  3. Set to Viewer (read-only)
Pro tip: Share ONE top-level folder → everything inside becomes accessible.

BigQuery

Grant the SA access to query data:

gcloud projects add-iam-policy-binding YOUR_PROJECT_ID --member="serviceAccount:gated-info@YOUR_PROJECT_ID.iam.gserviceaccount.com" --role="roles/bigquery.user"
gcloud projects add-iam-policy-binding YOUR_PROJECT_ID --member="serviceAccount:gated-info@YOUR_PROJECT_ID.iam.gserviceaccount.com" --role="roles/bigquery.dataViewer"

Or via Console: IAM → find the SA → Edit → Add roles.

Gmail OAuth2

SA cannot access Gmail. Gmail requires user consent (OAuth2).

  1. Go to Credentials
  2. "+ Create Credentials" → OAuth Client ID → Desktop app
  3. Download client_secret_xxx.json
  4. Set OAuth consent screen to Production (avoids 7-day token expiry)
node --experimental-strip-types bin/gated-info.ts auth gmail --client-secret ~/Downloads/client_secret.json

Opens browser → log in → authorize → refresh token saved to Keychain. This grants gmail.readonly scope.

Sending emails

Requires a separate token with gmail.send scope (reuses the same OAuth Client ID):

node --experimental-strip-types bin/gated-info.ts auth gmail --send

Subject and body are RFC 2047/base64 encoded — non-ASCII characters (Cyrillic, emoji, etc.) are fully supported.

Google Calendar

  1. Google Calendar → Settings → calendar → Share with specific people
  2. Add SA email → See all event details

Step 4: Connect to gated-info

node --experimental-strip-types bin/gated-info.ts auth google --service-account ~/Downloads/gated-info-key.json
node --experimental-strip-types bin/gated-info.ts scan
Done. You can delete the key file — it's now stored in macOS Keychain.

Available Google Services

Checklist

Pick what you need:

  • Google Drive — enable API, share folder with SA
  • Google Sheets — enable API (Drive share covers access)
  • Google Docs — enable API (Drive share covers access)
  • BigQuery — enable API, grant bigquery.user + bigquery.dataViewer roles
  • Gmail — enable API, create OAuth Client ID, run auth gmail
  • Google Calendar — enable API, share calendar with SA

After enabling, run:

node --experimental-strip-types bin/gated-info.ts scan

Troubleshooting

"403 Forbidden" — API not enabled, or SA doesn't have access.
1. Is the API enabled? → APIs & Services
2. Is the file/folder shared with SA email?
3. For BigQuery: does SA have IAM roles?
"401 Unauthorized" — Credential issue. Re-run:
gated-info auth google --service-account key.json
"Token expired" for Gmail — OAuth consent screen is in "Testing" mode.
OAuth consent screen → Publish App
"Key creation is not allowed" — Organization policy blocks SA key creation. If you manage the org:
gcloud resource-manager org-policies delete iam.disableServiceAccountKeyCreation --project=YOUR_PROJECT_ID

Or via Console: Org Policies → find iam.disableServiceAccountKeyCreation → Not enforced. Re-enable after downloading the key.

BigQuery "Access Denied" — SA needs roles at project level. See Step 3: BigQuery.

gated-info — MCP server for auth-gated sources. Credentials in macOS Keychain, not on disk.
